Markdown学习笔记

发表于 2019-10-18 更新于 2019-10-26 分类于 工具 ， 写作 ， Markdown 阅读次数： Valine：

Markdown（MD）是一种一种流行的轻量级标记语言，很多网站和博客系统都支持，比如Github、简书和Hexo。对比HTML它更加易读易写，适合用来快速码字和简单的排版。

本篇笔记仅适合由HTML基础的同学阅读

历史与不同的标准

John Gruber于2004年与Aaron Swartz合作用Perl语言编写了第一个markdown-to-html转换器，创造了Markdown语言。由于其易用性和可读性，在网络上广泛被使用。

但是由于最初的的语法规范比较模糊，而且人们对于一些基本语法之外的功能有所需求——比如表格、脚注等，以至于Markdown被实现出很多不同版本。

为了解决这个问题Jeff Atwood和John MacFarlane等人启动了标准化工作，致力于建立一个标准、明确的Markdown语法规范。由于Gruber反对在这一工作中继续使用“Markdown”这个名字，其被更名为 CommonMark。

2017年，Github发布了CommonMark的严格超集 GitHub Flavored Markdown（GFM） 。它在CommonMark的基础上扩展了一些功能，比如表格、脚注、删除线、自动链接、任务清单等。

本文参考最初的语法规范所写，并且加上被GFM扩展的语法。如果你想学习CommonMark标准语法且有英语基础的话，我强烈推荐官方发布的这个简单易懂的交互式教程，10分钟就能学完。

段落和换行

Markdown段落由一个或多个文本行组成，它的前后要有一个或以上的空行来换行（看不见东西就算空行，有Space和Tab也不影响是不是空行）

注意：写MD的时候，普通段落不建议用Space或Tab缩进，因为四个空格会被识别成代码块！

段落一

段落二

在段落的后面使用两个Space或者一个反斜杠\然后Enter或者可以在段落内强制换行。

段落第一部分
段落第二部分
段落第三部分

Hexo的Markdown用法不同，想强制换行直接Enter就行✌

不过写的时候还是推荐用标准的语法，不然你在其他地方写（比如我写MD的VS Code）Markdown的时候可能会不适应

被兼容的HTML标签

Markdown 本身没有一些样式，但是可以通过兼容的HTML标签来实现。

不在 Markdown 涵盖范围之内的标签，都可以直接在文档里面 用 HTML 撰写。不需要额外标注这是 HTML 或是 Markdown；只要直接加标签就可以了。

要制约的只有一些 HTML 区块元素――比如 <div>、<table>、<pre>、<p> 等标签，必须在前后加上空行与其它内容区隔开，还要求它们的开始标签与结尾标签不能用制表符或空格来缩进。Markdown 的生成器有足够智能，不会在 HTML 区块标签外加上不必要的 <p> 标签。

列表

Markdown支持无序列表和有序列表

  1. 无序列表可以用 + - * 来标记

    • 项目一
    • 项目二
    • 项目三

  2. 有序列表可以用数字句号空格标记

    1. 项目一
    2. 项目二
    3. 项目三

    数字的顺序不是重点，输出都一样,但是还是推荐按顺序

    1. 项目一
    2. 项目二
    3. 项目三

两个列表项目之间可以用空行分开，列表项目会被

包起来，但是效果没有什么不同

列表还可以嵌套多个段落、列表、代码块、引用块等。嵌套的时候注意要与列表项目第一段进行缩进对齐，才能识别为同一个列表项目。

引用块

如果你想表示一些内容引用自其他地方，你可以用>标记区块引用。和列表一样，区块引用可以嵌套，区块内也可以使用其他Markdown语法

勇士总冠军 —— 鲁迅

我没说过这句话 —— 鲁迅

你不必每一行都用>标记，如果你想偷懒也可以只在段落的开头写，直到空行前都是包含在区块引用内。

代码块

如果你想插入一些代码，你可以用代码块来标记。

  1. 行内代码用一对反引号`标记

    Javascript可以用 console.log("") 来输出

  2. 代码块用一对三个反引号```标记，如果想要获得某些Markdown渲染器的语法高亮，可以选择在第一个```后面加上语言的名称

  3. 除此之外代码块还可以用四个空格或制表符缩进来表示，一个代码区块会一直持续到没有缩进的那一行

    var str = "hello world";
console.log(str);

  4. 如果你想在代码块里面加入反引号 ` ，你只需要保证外面那层的反引号数量多于里面的

链接（含GFM扩展）

Markdown支持两种链接的标记方法：行内式和参考式

  1. 行内式：

    [显示的内容](地址)
    或者
    [显示的内容](地址 "地址标题")
    或者
    <地址>
    或者GFM扩展的自动连接
    链接

    鼠标移动到链接的时候会显示地址标题

    百度
    谷歌
    https://www.bing.com
    https://duckduckgo.com

  2. 参考式

    先写链接 [显示的内容][标记] 然后再文件任何地方加入链接的内容定义 [标记]: 地址 "地址标题"
    CommonMark标准的渲染器需要在这两者之间加入空行

    注意标记的匹配，不区分大小写。地址标题可以用单引号、双引号或者括号包起来，也可省略不写。地址也可以用尖括号包起来

    Facebook
    Github

    你甚至可以省去标记，把显示的内容当作标记

    代码块

关于URL地址的使用技巧可以参考MDN的教程

图片

图片和链接一样的标记也有行内式和参考式，我只展示行内式，可以以此类推出参考式

Markdown暂时还不支持设置图片的长宽，你可以用html的<img>来代替

脚注（Pandoc扩展）

很多插件或者Markdown引擎扩展了脚注的标记，比如由CommonMark制定者之一John MacFarlane开发的万能文本转换器Pandoc

脚注一般类似于链接的参考式，也有插件支持行内式，但是不常用所以这里不提

脚注的语法为 [^标记] 然后再文章的任何地方（推荐在文件末尾）加入脚注的定义 [^标记]: 定义内容 ，定义内容可以由很多行，需要至少四个空格或一个制表符的缩进对齐。

表格（GFM扩展）

你可以用管道符 | 和减号 - 来创建表格。其中管道符用于分割每一列，减号用来标记列的标题。管道符不需要完美对其，但是标题至少要用三个减号标记。

还可以设置对齐方式：

  1. :--- 左对齐
  2. :---: 居中
  3. ---: 右对齐
